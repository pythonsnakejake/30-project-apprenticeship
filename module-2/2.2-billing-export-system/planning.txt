3. SERIALIZATION LINES: (Why does calling `file.write("LINE ONE")` followed immediately by `file.write("LINE TWO")` result in a broken text format 
inside the physical text file? What hidden character string must you explicitly add to the end of your text sentences to keep them separated?)
Calling file.write() twice back-to-back results in a broken layout because the method does not automatically add a newline character at the end of your string. 
We must add a new line character "\n" to ensure that each line of data is actually written on a new line instead of it being one continues string of text.
