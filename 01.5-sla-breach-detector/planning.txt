   - Look at the calculation: `duration = shipped_hour - received_hour`.

3. EXCEPTION HANDLING LOGIC: (Explain in your own words why catching generic exceptions using a blanket `except:` statement is considered a dangerous, 
low-quality software engineering habit compared to catching specific exceptions like `except TypeError:`.)

Sometimes we may get errors that we can't, or shouldn't recover from, and allowing a program which requires user input could have potentially dangerous
consequences. It would also make debugging our program harder as we would not know the specific error that is causing our code to fail. 
A blanket except: block will also capture keyboard interrupts (Ctrl+C) or system termination signals. That means if a developer tries to manually stop a stuck program, 
a blanket except: block will intercept that signal and refuse to let the program shut down!
